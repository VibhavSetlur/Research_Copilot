# Autonomous Data Research Co-Pilot — Project Template

**Copy this folder to start any research project. The only document you write by hand is `docs_input/research_brief.md`. All scripts, data validation, analysis, figures, tables, and the interactive dashboard are generated by the agent pipeline.**

---

## Quick Start

### Step 1 — Fill the research brief

Open `docs_input/research_brief.md` and complete all 9 sections (study title, domain, hypotheses, dataset, variable taxonomy, design context, exclusion criteria, deliverables, known issues). Vague entries produce downstream failures.

### Step 2 — Drop your data

Place all raw data files in `data_raw/`. Do not rename, transform, or pre-process — the agents handle everything.

### Step 3 — Pick your AI tool and run

You can interface with this template using any agentic AI system. A unified orchestrator script (`scripts/research_runner.py`) maintains prompt state, outputs fully-hydrated agent prompts, and auto-syncs IDE-specific configurations.

#### General Agentic / Copy-Paste Workflow
For generic LLMs (Claude Projects, ChatGPT, Gemini, etc.):
1. Generate the fully hydrated prompt (including guardrails and current file context) for the active phase:
   ```bash
   python3 scripts/research_runner.py prompt init
   ```
2. Copy the terminal output and paste it into the AI chat window.
3. Once the AI completes the task, update the system status (optional):
   ```bash
   python3 scripts/research_runner.py status
   ```

#### Terminal-Capable / Autonomous Agents (OpenCode, Antigravity, Codex)
OpenCode and Antigravity slash commands are dynamically generated and linked directly to the python orchestrator:
```bash
/research-init
/research-route
/research-scaffold
/research-analyze
/research-compile
/research-audit
```
To check active status: `/research-status`.

#### Code Editors (Cursor, GitHub Copilot)
The rules are dynamically synced to IDE configuration files. Simply use the slash prompts in Cursor or Copilot Chat:
- **Cursor**: Custom instructions are synced to `.cursorrules` and `.cursor/CURSOR_INSTRUCTIONS.md`. Type phase commands in Cursor Chat.
- **GitHub Copilot**: Custom instructions are synced to `.github/copilot-instructions.md`. Use `@workspace` to target commands.

To synchronize configs manually at any time:
```bash
python3 scripts/research_runner.py sync
```

## Pipeline Overview

Each phase depends on all prior phases. Do not skip.

```
Phase 1: /research-init      → Deep profiling, domain + data type classification, causal map, power analysis
Phase 2: /research-route     → Method routing, live literature search, DOI-cited dependency manifest
Phase 3: /research-scaffold  → Domain-calibrated directories, pandera validation, data dictionary
Phase 4: /research-analyze   → Domain-specific transforms, assumption-pivot engine, FDR correction, publication figures
Phase 5: /research-compile   → IMRAD report, publication tables, 5-tab interactive dashboard
Phase 6: /research-audit     → Cold-start reproducibility + 9-dimension compliance sweep
```

## Key Design Principles

1. **Domain-aware from the start** — Phase 1 classifies the domain and data type; all downstream decisions adapt.
2. **Live literature grounding** — Phase 2 searches live academic databases for comparable studies and cites them with DOIs.
3. **Contracts over conventions** — Every agent has explicit INPUT SPEC, OUTPUT SPEC, and FAILURE PROTOCOL.
4. **Assumption-pivot as first-class output** — The methods log IS the Methods section. Every deviation is documented with a citation.
5. **Effect sizes and CIs are mandatory** — No p-value is reported without a test statistic, degrees of freedom, effect size, and 95% CI.
6. **FDR correction for multiple comparisons** — Applied automatically when >3 hypotheses are tested.
7. **9-dimension release gate** — Environment, pipeline execution, hashes, pivot coherence, leakage, causal language, statistical reporting, figure completeness, dashboard execution.
