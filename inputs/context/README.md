# context/ — Background Materials

Add any supplementary materials that help the system understand your research context.

## What to Include

| Type | Examples |
|------|----------|
| **Abstracts** | Paper abstracts you've collected |
| **Notes** | Research notes, literature summaries |
| **Links** | URLs to datasets, papers, or resources |
| **Background docs** | Theoretical frameworks, prior reports |
| **Codebooks** | Variable definitions, survey instruments |
| **Protocols** | Study protocols, IRB documents |

## File Formats

- `.md` — Markdown notes
- `.txt` — Plain text
- `.pdf` — Documents
- `.url` — Link files (one URL per line)

## Naming Convention

Use descriptive names: `theoretical_framework.md`, `survey_codebook.txt`, `useful_links.url`.

## Example: useful_links.url

```
https://doi.org/10.1038/xxx
https://osf.io/xxx
https://github.com/xxx/dataset
```

## Example: survey_codebook.txt

```
variable_name | label | type | range | notes
age | Age in years | numeric | 18-95 | Self-reported
income | Annual income | numeric | 0-500000 | In USD, top-coded
gender | Gender | categorical | M/F/Other | Self-identified
```
