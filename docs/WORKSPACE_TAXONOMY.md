Every Research OS project follows a strict, reproducible directory structure. This document explains every folder and file.

## `inputs/` ‚Äî Immutable Source Data

Write-protected at the tool level. Tools will raise `WriteProtectedError` if they attempt to modify anything under `inputs/`.

```
inputs/
‚îú‚îÄ‚îÄ raw_data/            # Source data files (CSV, Parquet, JSON, etc.)
‚îÇ   ‚îî‚îÄ‚îÄ survey.csv
‚îú‚îÄ‚îÄ literature/          # PDF papers (keep original filenames)
‚îÇ   ‚îú‚îÄ‚îÄ smith2024.pdf
‚îÇ   ‚îî‚îÄ‚îÄ jones2023.pdf
‚îú‚îÄ‚îÄ context/             # Supporting context files
‚îú‚îÄ‚îÄ literature_index.yaml   # Auto-generated: maps filename ‚Üí citation_key, sha256, verified
‚îî‚îÄ‚îÄ intake.md               # Auto-generated: research question, domain, SHA-256 table of all input files
```

### `intake.md`

Generated by `regenerate_intake()` (called during `init` and every `mem.regenerate.intake`). Contains:
- Project name, domain, depth
- Research question
- Keywords
- Table of every input file with SHA-256 hash and size

### `literature_index.yaml`

Generated by `update_literature_index()`. Maps each PDF filename to:
- `citation_key`: auto-derived from stem (e.g., `smith2024`)
- `sha256`: content hash for integrity verification
- `verified`: boolean flag for citation verifier

### Numbered Experiment Folders (`01_`, `02_`, ...)

Created by `sys.branch.create`. Each folder is a self-contained step.

**`README.md`** contains:
- **Goal**: What this experiment aims to discover
- **Input Data**: Files and sources used
- **Methods Used**: Statistical tests, transforms, models
- **Expected Output**: Predicted results before execution
- **Actual Output**: Results after execution
- **Next-Step Decision**: proceed / branch / dead-end

**`conclusions.md`** contains:
- Summary of findings
- Key metrics and effect sizes
- Next steps

### `analysis.md`

The "state machine" log. Updated by `sys.analysis.log` after every significant step.

Contains:
- **Mermaid workflow diagram**: auto-updated with status-colored nodes (green=complete, yellow=running, grey=planned, red=failed, dashed red=dead-end)
- **Chronological log entries**: `[timestamp] step_name: description`

### `methods.md`

Append-only. Every tool that runs a statistical test, data transformation, or literature retrieval must append a structured entry via `mem.methods.append`:
```markdown
## 2026-05-22 14:03:00
- **Method**: Welch t-test
  - **Parameters**: equal_var=False, alpha=0.05
  - **Tool**: tool.statistical.test
  - **Citation**: student1908
```

### `citations.md`

Running bibliography. Generated from `literature_index.yaml` by `mem.citations.generate`. Each entry has a `verified` flag:
```markdown
### `smith2024`
  - **File**: smith2024.pdf
  - **SHA-256**: `a1b2c3d4e5f6...`
  - **Status**: ‚è≥ Pending verification
```

## `.os_state/` ‚Äî Internal State (DO NOT TOUCH)

Managed entirely by Research OS. Manual edits may corrupt the state machine.

```
.os_state/
‚îú‚îÄ‚îÄ state_ledger.yaml      # Primary source of truth (YAML)
‚îú‚îÄ‚îÄ state_ledger.json      # JSON backup for compatibility
‚îú‚îÄ‚îÄ manifest.json          # Workspace manifest
‚îú‚îÄ‚îÄ checkpoints/           # Workspace snapshots (created by sys.checkpoint)
‚îÇ   ‚îú‚îÄ‚îÄ before_model_training/
‚îÇ   ‚îÇ   ‚îî‚îÄ‚îÄ checkpoint_manifest.json
‚îÇ   ‚îî‚îÄ‚îÄ ...
‚îî‚îÄ‚îÄ replay_logs/           # Session replay for audit
```

### `state_ledger.yaml`

The single source of truth. Contains:
- `project_id`, `project_name`
- `current_branch`, `step`, `pipeline_stage`
- `branches`: all branches with status, hypothesis, creation time
- `checkpoint_history`: ordered list of snapshots
- `updated_at`: ISO timestamp of last change

Updated atomically (temp file + rename) after every tool execution. Before/after diffs logged to `workspace/logs/state_changes.log`.

## Conventions Summary

| Rule | Enforcement |
|------|-------------|
| `inputs/` is immutable | `WriteProtectedError` at tool level |
| Experiments are numbered `01_`, `02_`, ... | `sys.branch.create` auto-numbers |
| Every experiment has a `README.md` | Created by `create_numbered_experiment()` |
| `methods.md` is append-only | `mem.methods.append` appends, never overwrites |
| `analysis.md` contains Mermaid diagram | `sys.analysis.log` updates diagram + log |
| State changes are logged | `save_state()` writes diff to `state_changes.log` |
| Files have SHA-256 checksums | Every tool response includes checksums |
