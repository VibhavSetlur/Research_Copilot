# Example 1: Sleep Quality Analysis

This directory is a **fully populated Research Copilot workspace** demonstrating a straightforward statistical analysis workflow. 

It represents the end-state of a project where a researcher has:
1. Initialized the workspace (`rcp init`).
2. Placed raw data in `00_inputs/raw_data/sleep_quality.csv`.
3. Ingested the data for vector search (`rcp ingest`).
4. Used the `ResearchEngine` to explore the intent: *"Explore whether the intervention improves sleep scores and summarize key findings."*
5. Generated analytical outputs (a Welch's t-test summary, a CSV of group means, and a bar chart).

## Exploring the Workspace

- **`00_inputs/`**: Contains the raw and processed data.
- **`01_workspace/`**: Contains intermediate logs, engine routing decisions, and the vector search cache (`.research/cache`).
- **`02_experiments/`**: Contains the output artifacts generated by the system in `exp_001_baseline/outputs/`. Check here for the generated `summary.json`, `group_means.csv`, and the generated figure.
- **`03_synthesis/`**: Contains the final state ledger (`state_ledger.json`) tracking the completion of the project.
- **`RESEARCH_OVERVIEW.md`**: The initial problem statement and data dictionary the researcher provided.

*Note: This is not a "runnable script" example, but a pristine example of the artifact structure produced by the Research Copilot system operating at production quality.*
